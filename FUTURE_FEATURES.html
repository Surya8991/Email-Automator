1. Deliverability

HighM

SMTP warmup mode

Ramp daily send count gradually (e.g. 10 → 50 → 100 → 200) for new credentials so a fresh sender doesn't trigger Gmail's anti-spam.

Add a warmup_started_at per-identity column; scheduler-tick honors the cap.

HighM

Recipient-TZ send window

Only fire sends between 09:00–18:00 in the recipient's local time. Push outside-window sends to the next valid hour. Reads timezone from contact custom field with a per-platform default.

HighM

Subject A/B winner picker

After N sends across a template's subject/subjectB, auto-promote the higher-reply-rate variant. Show "promoted by X% reply rate at sample=Y" in analytics.

MedS

Pre-flight spam-trigger lint

Before send, scan body for words / patterns that score on SpamAssassin or Apple Mail filters (ALL CAPS, "FREE!!", excessive exclamation, suspicious links). Surface as a warning chip in the editor.

MedS

One-click DKIM helper

If diagnostic finds no DKIM record, show the recommended TXT entry and a copy button. Reduces manual lookup of provider docs.

MedM

Per-recipient send-cap memory

Hard-block re-sending to the same address within X days even if the user manually retries. Surfaces the existing duplicate-send warning as an actual block with override.

LowL

Per-IP rotation

Multiple SMTP credentials per user → scheduler-tick round-robins to spread load and reputation risk.

2. AI & intelligence

HighM

AI Improve diff view

Old / new side-by-side with Keep / Revert. Currently the rewrite silently overwrites, which is hostile to iteration.

HighM

Reply-classifier

Auto-tag inbound replies as interested / needs follow-up / out of office / not interested / unsubscribe via a cheap LLM call. Feeds into the campaign stop-on-reply logic so OOO replies don't end the sequence.

MedM

Recipient enrichment from email signature

When a contact replies, parse their signature for title / company / phone and offer to update the contact record.

MedS

Tone match across campaign steps

If step 1 was sent in a "friendly" tone, suggest the same tone for follow-ups so the sequence reads as one voice.

MedM

Best-time-to-send predictor

Per-recipient (or per-platform) best send hour learned from open-time data. Currently the heatmap visualizes; this would act on it.

LowL

Self-hosted LLM swap

Drop-in Ollama / vLLM backend so users worried about Groq data residency can swap to a local model. Service interface is already abstracted.

5. Mobile & offline

Offline-first PWA

Service-worker cache of the last contacts + drafts so the app loads on a flaky connection. Reads stay live, writes queue + retry.

HighM

Pull-to-refresh on lists

Native-feeling refresh on /drafts, /contacts, /campaigns. Especially valuable on mobile where the desktop refresh shortcut isn't available.

MedS

LowL

Share-to-EA from any app

Presence pill is shipped; next step is conflict-free co-editing of campaign step copy. Yjs + a small WebSocket.

LowS

Toggleable footer link on outgoing emails. Off by default; off feels right for a self-hosted tool.

10. Job Tracker — next phase

Shipped (2026-06-06): 14 adapters · normalization · cross-board dedup · marketing-first sort · seniority badges · 15-day ingest window · 2 000-lead limit · delete-by-age · cancel button · restore to New/Saved · rel=noopener noreferrer security fix.

HighL

Pipeline kanban (Huntr-style)

Horizontal columns: New → Saved → Applied → Screen → Interview → Offer → Rejected. Drag-to-move via @dnd-kit/core or a CSS-only "Move to →" dropdown for v1. Per-card: title chip, company, salary chip, follow-up countdown, last-action date.

Requires status enum expansion in the DB schema.

HighM

Resume ↔ JD match score (Jobscan-style %)

Tier 1: pure-JS TF-IDF overlap between your resume text and the JD → color-coded chip (green ≥75 / amber 50–74 / red <50). Free, no tokens. Tier 2 (opt-in): Groq llama-3.1-8b-instant returns {score, matchedSkills, missingSkills} cached per (resumeHash, leadId).

New user_resumes table + /profile upload card.

HighM

Follow-up date + overdue pill

New nextFollowUpAt column. When a lead enters Saved, default to seenAt + 3 days. Amber pill when due, red when overdue. Click → date picker. Cron writes one events row per user per day so the topbar bell can surface overdues.

HighM

Notes on lead detail dialog

Inline <textarea> auto-saved on blur. Backed by a notes column + setJobLeadNotesAction rate-limited 60/min. Makes Saved leads actually useful as a personal CRM without leaving the app.

MedL

Browser extension — "Save to Tracker" (Simplify-style)

Manifest V3 extension. Content script detects <script type="application/ld+json"> on any JD page, surfaces a floating "Save to Job Tracker" button. Hits new /api/jobs/capture route (auth via API key with scope jobs:write). Fallback: select JD text → Groq extractor.

Works on LinkedIn / Indeed / Glassdoor JD pages without any server-side scraping.

MedS

CSV import of leads

Mirror the contacts import pattern. Route /api/jobs/import with SSE progress. Accepts the same columns the CSV export already emits — migration path for Huntr / Teal / Simplify users.

MedS

Source-yield sparkline on Sources tab

Tiny inline SVG (no charting library) showing leads-per-day over the last 7 days per source. Lets you spot which boards are pulling weight vs going dry.

LowS

JSON-LD extraction-path telemetry

New extractionPath enum column (adapter | jsonld | groq). Widget on /admin → System: 7-day breakdown per host. Target: adapter+jsonld >80% so the Groq bill stays cheap.
