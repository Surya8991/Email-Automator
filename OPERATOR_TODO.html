1 URGENT — apply prod migrations

Why your prod is 500'ing. The latest pushes added eight new database tables: progress_events, companies, email_identities, campaign_step_variants, saved_views (for /contacts saved filter combos), job_sources + job_leads (new — for the Job tracker), plus the contact_form audit rows. Until you run the migrations against your Turso DB, every page that touches one of these tables crashes — that's what's behind the /settings, /companies, and /contacts (saved-views read) errors you've been seeing. Defensive .catch wrappers degrade gracefully but you still need to migrate to unlock the features themselves.

Step 1.1 — Run the migrations

From your laptop, with the credentials Vercel uses for prod:

cd C:\Coding\Email-Automator
DATABASE_URL='libsql://YOUR-DB.turso.io' TURSO_AUTH_TOKEN='eyJ...' npm run db:migrate

Where to find those two values:

Expected output:

[db] migrated (libsql): libsql://YOUR-DB.turso.io

Step 1.2 — Redeploy

Once the redeploy finishes, all the pages that were 500'ing will load. The defensive .catch wrappers we added earlier mean if a single backend call still fails after the migration, the page degrades to an empty-state instead of crashing.

2 Verify it worked

Sign in once the redeploy is done, then open each URL. They should all render.

If any of these still 500, open Vercel → Deployments → latest → Logs and filter to error. Look for lines starting with [settings], [companies], [contacts], [campaigns], or [layout] — those are the defensive console.error markers I added so you see exactly which read failed.

3 Recommended hardening soon

3.1 — Set ENCRYPTION_KEY

SMTP passwords and Groq API keys are AES-GCM encrypted at rest. The key derives from ENCRYPTION_KEY if set, else from AUTH_SECRET. Setting it explicitly decouples the two so rotating one doesn't invalidate the other.

openssl rand -base64 32

3.2 — Confirm CRON_SECRET is set

Without CRON_SECRET, the /api/cron/tick endpoint is open to the public in production. Anyone who finds the URL can trigger the scheduler to send emails on behalf of all your users.

3.3 — Set HSTS_APEX=true only if you have an apex domain

Do NOT set this on a *.vercel.app host. The strong HSTS form (includeSubDomains; preload) would force HTTPS on every other app on the shared vercel.app zone, and the preload directive is invalid from a non-apex eTLD+1. Only enable when your prod URL is your-app.com.

4 Optional: GitHub OAuth 10 min

Adds a Continue with GitHub button on /login alongside Google + magic link.

5 Optional: Redis rate limiter 15 min

The current in-memory rate limiter is per-Lambda instance — on Vercel, an attacker can hit the same endpoint from one IP and the bucket resets for each warm Lambda. Wire Upstash Redis to fix that.

Note: the implementation that consumes these vars is scaffolded but not yet wired on every route. The warning goes away; the actual swap is a follow-up sprint.

6 Optional: CSP nonce audit first

Do not enable until you've audited every inline script in the app. The middleware ships ready, but enabling it without first threading the nonce through Next's hydration runtime, Recharts, sonner toast styles, etc. will white-screen the app.

8 Troubleshooting

How to read Vercel logs

  1. Vercel → your project → Deployments
  2. Click the latest Ready deployment
  3. Click the Logs tab (or click any specific function)
  4. Filter to error

What the bracketed log lines mean

Log prefix Means Fix
[settings] getMany failed Settings table read failed Check Turso connection; retry
[settings] listIdentities failed Migration 0006 not applied to Turso Run the migrate command in section 1
[companies] listCompanies failed Migration 0006 not applied Same as above
[contacts] getCompanyByName failed Migration 0006 not applied Same as above
[campaigns] listVariants failed Migration 0006 not applied Same as above
[sse] readLatest failed Migration 0005 not applied (progress polling falls back to SSE) Same as above
[contact] sendMail failed (non-fatal) SMTP misconfigured; contact form still wrote to audit log Check SMTP_USER / SMTP_PASS in Vercel env
[rate-limit] Running on a multi-instance platform… Per-Lambda buckets warning Wire Upstash (section 5)
[cron/tick] CRON_SECRET is not set Critical: scheduler endpoint is open to the public Set CRON_SECRET in Vercel + GitHub Actions secrets (section 3.2)

Common errors and fixes

Symptom Likely cause
/settings or /companies returns 500 Migrations 0005 + 0006 not applied to Turso
Sign-in works but redirects to a blank page Stale build chunk — hard refresh (Ctrl+Shift+R)
"Magic link 500" Wrong Gmail App Password — regenerate at myaccount.google.com/apppasswords
Continue with GitHub doesn't appear on /login GITHUB_ID or GITHUB_SECRET missing; redeploy needed after adding
Contact form submits but no audit row Migration 0006 not applied (audit table is older but the action runs through the same connection)
AI buttons return "Groq API key not set" Set GROQ_API_KEY in Vercel env, OR sign in and set the per-user key in Settings → AI

Need to ask something specific? Sign in, open /contact, and the message lands in your own audit log + optionally your inbox. Or file a GitHub issue — faster turnaround on deployment problems.
