📬 Email Automator — your setup checklist

A one-page walkthrough of everything you need to do to ship this on Vercel. Tick the boxes as you go — your browser remembers them across page loads.

What's in this page

  2. Generate AUTH_SECRET + CRON_SECRET (1 min)

  5. Get a Groq API key (2 min) — optional

1 Prereqs

2 Generate AUTH_SECRET + CRON_SECRET

You need two long random strings. Run this twice on your laptop:

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Save both somewhere temporary — you'll paste them into Vercel later.

3 Gmail App Password for SMTP required

Magic-link login and every outgoing email goes through SMTP. For Gmail:

  1. Visit myaccount.google.com/apppasswords
  2. App: Mail, Device: Other (Email Automator) → Generate
  3. Copy the 16-character password (no spaces). Save it temporarily.
⚠️ App Passwords require 2-Factor Authentication on your Google account. Enable 2FA first if it's not on.

4 Create the Turso database required for Vercel

Vercel is serverless — the local SQLite file won't persist. Turso gives you a hosted SQLite-compatible DB. The app's dual driver detects the URL and uses Turso automatically — no code changes needed.

4a. Install the Turso CLI

# macOS / Linux
curl -sSfL https://get.tur.so/install.sh | bash

# Windows (PowerShell)
irm get.tur.so/install.ps1 | iex

4b. Create the DB and grab the URL + token

turso auth signup        # one-time
turso db create email-automator
turso db show email-automator --url            # → libsql://email-automator-<org>.turso.io
turso db tokens create email-automator         # → eyJ…   (this is TURSO_AUTH_TOKEN)

4c. Apply migrations to the new DB (one-time, from your laptop)

DATABASE_URL=libsql://email-automator-<org>.turso.io \
TURSO_AUTH_TOKEN=eyJ… \
npm run db:migrate

5 Get a Groq API key optional

Powers the AI Improve button in the template editor. Skip if you don't want that feature. Free tier covers light usage.

  1. console.groq.com → API Keys → Create
  2. Save the gsk_… key
💡 Tip: you can also set this per-user from Settings → AI inside the running app, so you don't have to set the env var globally.

6 Set up Google OAuth optional

Adds a "Continue with Google" button on the login page. Magic link still works either way.

  1. Open Google Cloud → Credentials
  2. Create OAuth Client ID → Web application
  3. Authorized redirect URI: https://your-vercel-url.vercel.app/api/auth/callback/google
  4. Copy the Client ID + Client Secret — they go in Vercel env vars below

7 Deploy to Vercel

7a. Push your repo to GitHub (if not already)

git remote -v          # confirm origin points at your fork
git push origin main

7b. Import the project in Vercel

  1. vercel.com/new → Import Git Repository → pick this repo
  2. Root Directory: web (important — the Next app lives under web/, not the repo root)
  3. Framework Preset: Next.js (auto-detected)
  4. Don't click Deploy yet — set env vars first

7c. Paste these env vars in Vercel → Settings → Environment Variables

# ── required ─────────────────────────────────────────────
AUTH_SECRET=<the first secret from step 2>
APP_URL=https://your-app.vercel.app
NEXTAUTH_URL=https://your-app.vercel.app
DATABASE_URL=libsql://email-automator-<org>.turso.io
TURSO_AUTH_TOKEN=<from step 4b>
CRON_SECRET=<the second secret from step 2>
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=you@gmail.com
SMTP_PASS=<the Gmail App Password from step 3>
EMAIL_FROM=Your Name <you@gmail.com>
ADMIN_EMAILS=you@gmail.com

# ── optional ─────────────────────────────────────────────
GOOGLE_CLIENT_ID=<from step 6, or leave empty>
GOOGLE_CLIENT_SECRET=<from step 6, or leave empty>
GROQ_API_KEY=<from step 5, or leave empty — can set per-user later>
GROQ_MODEL=llama-3.3-70b-versatile
DAILY_SEND_LIMIT=50
TIMEZONE=Asia/Kolkata
🚨 Do NOT set ALLOW_DEV_SIGNIN=true in production. It bypasses authentication for any email on DEV_BYPASS_EMAILS.

7d. Deploy

Click Deploy. ~2 minutes. The first visit hits app/(app)/layout.tsx which auto-seeds 20 starter templates on your first sign-in.

8 Post-deploy checklist

9 Alternative: self-host on Linux

If you'd rather not use Vercel, you can run on any Linux box with Node 20+. SQLite file persists locally.

cp .env.example .env       # edit everything; leave DATABASE_URL as ./data/tracker.db
npm install --legacy-peer-deps
npm run build
npm run db:migrate
npm install -g pm2
pm2 start npm --name email-automator-web    -- start
pm2 start npm --name email-automator-worker -- run worker
pm2 save && pm2 startup

Then point your nginx/Caddy at localhost:3000. Back up ./data/ with rsync/restic.

Stuck? Open the running app and visit /guide for the full manual.
